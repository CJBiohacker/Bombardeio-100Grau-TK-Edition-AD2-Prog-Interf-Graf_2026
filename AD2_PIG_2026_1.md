## 1 Objetivo

Criar a interface gráfica do jogo 2D desenvolvido na AD1. O jogo deverá ser executado por meio de uma interface gráfica, substituindo a interação via terminal.

A lógica do jogo (regras, estado persistente, evolução, critérios de término, leitura e escrita de arquivos) deve ser a mesma implementada na AD1, sendo vedada a reimplementação dessas regras diretamente na interface gráfica.

A interface gráfica deverá atuar como uma camada de visualização e controle, comunicando-se com as classes do jogo já existentes.

### 1.1 Requisitos

- A aplicação deve possuir uma janela principal criada a partir da classe `Tk`;
- O redimensionamento da janela deve ser tratado adequadamente;
- Elementos do jogo devem ser representados visualmente por: cores, formas geométricas, ou símbolos desenhados no Canvas, de forma que seja possível identificar visualmente jogador, inimigos, obstáculos destrutíveis, obstáculos indestrutíveis, bombas e explosões;
- O jogador deve controlar o jogo por meio de eventos de teclado. O movimento do jogador é feito usando as teclas direcionais e as bombas devem ser posicionadas por meio de tecla específica;
- A interface deve ser atualizada a cada turno do jogo;
- Mudanças no estado do jogo devem ser refletidas visualmente no Canvas;
- O menu inicial do jogo deve ter:
  - Botão para iniciar nova partida;
  - Botão para encerrar a aplicação;
  - Informações do estado persistente atual.
- Com o jogo iniciado, devem ser exibidas as seguintes informações:
  - Quantidade de inimigos restantes;
  - Quantidade de turnos restantes;
  - Partida em execução, em pausa ou finalizada.
- Mensagens de erro ou aviso devem ser exibidas ao usuário por meio da interface gráfica;
- O encerramento da partida deve ser claramente sinalizado na interface.

## 2 O que Entregar

Deve ser enviado o código fonte em mídia digital e o programa deverá rodar em ambiente Linux com Python 3 (recomendável que rode em Python 2 também). O código fonte é composto pelo conjunto de códigos utilizados para que o programa execute com sucesso. Por exemplo, se você criou uma biblioteca que é importada para um programa principal, deverá enviar tanto o arquivo contendo o código da biblioteca quanto o arquivo contendo o código principal.

Não fornecemos código "gabarito" nesse curso, visto que não existe gabarito de programa. Cada pessoa implementa a mesma funcionalidade e resolve o mesmo problema de formas distintas. Contudo, o seu código deve atender a todos os requisitos listados neste enunciado e deve obedecer à descrição fornecida.
